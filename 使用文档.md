# Goosd Admin 系统使用文档

## 1. 系统简介

Goosd Admin 是一套小红书笔记推广订单管理系统，支持批量下单（阅读/点赞/曝光）、订单跟踪、退款管理、余额管理等功能。系统包含：

- **后端**：Node.js + Express + MySQL
- **前端**：Vue 3 + Element Plus（基于 Vben Admin 框架）

---

## 2. 环境要求

| 依赖     | 版本要求         |
| -------- | ---------------- |
| Node.js  | 18.x 或以上      |
| MySQL    | 8.0 或以上       |
| pnpm     | 9.x 或以上       |
| PM2      | 全局安装（可选） |

---

## 3. 部署与启动

### 3.1 数据库初始化

```bash
# 导入数据库结构
mysql -u root -p < database/goosd_admin.sql
```

### 3.2 后端配置

进入后端目录，复制并编辑环境配置：

```bash
cd el_node
cp .env.example .env
```

`.env` 关键配置说明：

| 变量名                      | 说明                                    | 示例                |
| --------------------------- | --------------------------------------- | ------------------- |
| `PORT`                      | 后端服务端口                            | `3000`              |
| `DB_HOST`                   | 数据库地址                              | `127.0.0.1`         |
| `DB_PORT`                   | 数据库端口                              | `3306`              |
| `DB_NAME`                   | 数据库名                                | `goosd_admin`       |
| `DB_USER`                   | 数据库用户                              | `goosd_admin`       |
| `DB_PASSWORD`               | 数据库密码                              | -                   |
| `JWT_SECRET`                | JWT 签名密钥，留空则每次启动自动生成    | -                   |
| `XHS_API_BASE_URL`          | 上游任务接口地址                        | `http://x.x.x:9101` |
| `XHS_API_TOKEN`             | 上游任务接口 Token                      | -                   |
| `TINYDATA_PREVIEW_API`      | 链接预览解析服务地址                    | -                   |
| `TINYDATA_PREVIEW_TOKEN`    | 预览服务 Token                          | -                   |
| `BATCH_CHECK_CONCURRENCY`   | 预校验外部接口并发数（默认 100）        | `100`               |
| `CORS_ORIGIN`               | 允许跨域的前端地址                      | `http://localhost:5777` |
| `SEED_DEMO_DATA`            | 是否初始化演示数据                      | `false`             |

### 3.3 安装依赖并启动

```bash
# 后端
cd el_node
npm install
# 开发模式
node src/server.js
# 生产模式（PM2）
npx pm2 start ecosystem.config.js
npx pm2 save
```

```bash
# 前端
cd webEL
pnpm install
# 开发模式
pnpm dev          # 默认端口 5777
# 生产构建
pnpm build        # 产物输出到 dist/ 并自动生成 dist.zip
```

### 3.4 前端生产部署

编辑 `webEL/.env.production`，将 `VITE_GLOB_API_URL` 改为实际后端地址：

```
VITE_GLOB_API_URL=http://你的服务器IP:3000/api
```

构建后将 `dist/` 目录部署到 Nginx 或其他静态文件服务器。

## 4. 角色与权限

系统有三个角色：

| 角色       | 中文名     | 权限范围                                           |
| ---------- | ---------- | -------------------------------------------------- |
| `super`    | 超级管理员 | 所有权限 + 可分配超级管理员角色                     |
| `admin`    | 管理员     | 查看所有用户数据、管理权限、审核退款和补单          |
| `user`     | 普通用户   | 只能查看和操作自己的订单、消费记录                  |

## 5. 功能模块说明

### 5.1 批量下单

**菜单位置**：批量下单

支持三种业务类型：
- **阅读**（view）：增加笔记阅读量
- **点赞**（like）：增加笔记点赞量
- **曝光**（impression）：增加笔记曝光量

**操作流程**：

1. 在文本框输入链接，每行格式为 `链接 数量`，用空格或 Tab 分隔
2. 点击「手动预校验」，系统会：
   - 解析每条链接，获取笔记信息（标题、作者、头像）
   - 校验链接有效性、重复性、冲突检测
   - 计算预估费用
   - 结果以流式方式逐条返回，按钮显示进度（如"校验中 3/100"）
3. 确认无误后，勾选「我已阅读并确认上述公告内容」
4. 点击「确认提交阅读/点赞/曝光」

**辅助功能**：
- **填充示例**：自动填入示例链接
- **一键删除问题链接并记录**：移除校验失败的链接，并保存问题链接记录
- **下单记录**：查看历史提交批次和订单详情
- **清空**：清空输入内容

**支持的链接格式**：
- 完整链接：`https://www.xiaohongshu.com/explore/笔记ID`
- 短链接：`http://xhslink.com/o/xxx`

### 5.2 下单记录

**菜单位置**：下单记录

查看所有已提交的批量订单记录，支持：
- 按批次查看订单列表
- 搜索链接、订单号、笔记 ID
- 按状态筛选（成功/失败）
- 查看每条订单的状态、数量、金额
- 点击「刷新」同步上游任务状态

### 5.3 批量查找

**菜单位置**：批量查找

跨批次搜索订单，支持：
- 输入多条链接批量查找
- 按日期范围筛选
- 查看匹配的订单详情

### 5.4 消费记录

**菜单位置**：消费记录

查看账户资金流水，包括：
- 订单扣费记录
- 退款入账记录
- 充值记录
- 每条记录显示变动前后余额
- 汇总统计（收入、支出、退款、净额）

### 5.5 退款记录

**菜单位置**：退款记录

管理退款申请：

**普通用户**：
- 查看自己的退款状态

**管理员**：
- 查看所有用户的退款申请
- 逐条审核退款（通过/拒绝）
- 按批次「一键通过」：批量通过同一批次的所有退款申请
- 按批次「一键拒绝」：批量拒绝并停止上游任务

**退款说明**：
- 申请退款时系统会自动停止上游任务
- 退款通过后，根据已完成量计算实际退款金额
- 退款拒绝后，订单变为终态，不再阻止同一笔记的新订单

### 5.6 补单列表

**菜单位置**：补单列表

管理批量补单请求：
- 用户可对已完成但未达标的批次发起补单申请
- 管理员审核补单请求
- 审核通过后系统自动创建补充订单

### 5.7 开放接口

**菜单位置**：开放接口

管理 Open API Key，支持外部系统对接：

**API Key 管理**：
- 创建新的 API Key（创建后只显示一次，请妥善保存）
- 查看已有 Key 的前缀和状态
- 吊销不再使用的 Key

**接口文档**：
页面内嵌接口文档，包含以下接口：

| 接口                        | 方法 | 说明             |
| --------------------------- | ---- | ---------------- |
| `/api/open/orders/preview`  | POST | 预校验订单       |
| `/api/open/orders/submit`   | POST | 提交订单         |
| `/api/open/orders/progress` | GET  | 查询订单进度     |
| `/api/open/orders/stop`     | POST | 停止订单任务     |

**鉴权方式**：请求头添加 `Authorization: Bearer <API_KEY>`

**提交订单示例**：

```json
{
  "target_type": "view",
  "agree_policy": true,
  "remark": "merchant_order=MO202605180001",
  "content": "http://xhslink.com/o/abc123 1000\nhttp://xhslink.com/o/def456 2000\nhttp://xhslink.com/o/ghi789 500"
}
```

- `target_type`：业务类型（view/like/impression）
- `remark`：必填，外部系统备注标识
- `content`：批量内容，多条链接用 `\n` 换行，每行格式为 `链接 数量`

### 5.8 权限管理

**菜单位置**：权限管理（仅 super/admin 可见）

管理系统用户和权限配置：
- **用户列表**：查看所有注册用户
- **角色分配**：为用户分配 超级管理员/管理员/普通用户 角色
- **余额管理**：调整用户账户余额（充值/扣费）
- **单价设置**：为每个用户单独设置下单单价
- **折扣配置**：设置用户折扣率
- **业务类型开关**：启用/禁用用户的 阅读/点赞/曝光 下单权限
- **账号状态**：启用/禁用用户账号

### 5.9 分析页

**菜单位置**：分析页

系统数据总览仪表盘：
- 订单总量和金额统计
- 用户排名（按消费金额/下单数量）
- 余额概览
- 近期活动记录

## 6. 常用操作指南

### 6.1 新用户注册与配置

1. 用户访问登录页，点击注册
2. 管理员登录 → 权限管理 → 找到新用户
3. 分配角色（通常为普通用户）
4. 设置初始余额（充值金额）
5. 设置单价和折扣（可选）
6. 开启需要的业务类型（阅读/点赞/曝光）

### 6.2 批量下单完整流程

1. 进入「批量下单」页面
2. 选择业务类型（阅读/点赞/曝光标签切换）
3. 在文本框粘贴链接列表，例如：
   ```
   http://xhslink.com/o/abc123 1000
   http://xhslink.com/o/def456 2000
   http://xhslink.com/o/ghi789 500
   ```
4. 点击「手动预校验」
5. 等待流式校验完成，检查右侧面板：
   - 有效行 / 失败行数量
   - 预计扣费金额
   - 可用余额是否充足
6. 如有失败链接，在右侧失败卡片中查看原因，修正或一键删除
7. 勾选公告确认
8. 点击提交按钮

### 6.3 退款处理流程

**用户发起退款**：
1. 在「下单记录」中找到对应批次
2. 点击批次退款申请（或单条订单退款）
3. 系统自动停止上游任务并提交退款申请

**管理员审核退款**：
1. 进入「退款记录」页面
2. 查看待处理的退款申请
3. 逐条审核或按批次一键通过/一键拒绝
4. 通过后系统自动计算退款金额并返还余额

### 6.4 外部系统对接

1. 进入「开放接口」页面
2. 点击「创建 API Key」，输入备注名称
3. 复制生成的 API Key（仅显示一次）
4. 调用方使用该 Key 请求开放接口
5. 查看页面内嵌的接口文档了解请求格式

## 7. 技术架构

```
┌─────────────────────────────────────────────────┐
│                   前端 (Vue 3)                   │
│         webEL / Element Plus / Vben Admin        │
│                   端口: 5777                     │
└──────────────────────┬──────────────────────────┘
                       │ HTTP / SSE
┌──────────────────────▼──────────────────────────┐
│                 后端 (Express)                    │
│              el_node / PM2 管理                   │
│                  端口: 3000                      │
├──────────────────────┬──────────────────────────┤
│    认证 (JWT)        │     业务服务               │
│    权限控制          │     批量下单/退款/补单      │
│    请求日志          │     余额管理/消费记录       │
└──────────────────────┼──────────────────────────┘
                       │
          ┌────────────┼────────────┐
          ▼            ▼            ▼
    ┌──────────┐ ┌──────────┐ ┌──────────┐
    │  MySQL   │ │ 上游任务 │ │ Tinydata │
    │  数据库  │ │   API    │ │ 预览服务 │
    └──────────┘ └──────────┘ └──────────┘
```

### 数据库核心表

| 表名                         | 说明               |
| ---------------------------- | ------------------ |
| `users`                      | 用户账号           |
| `roles` / `user_roles`       | 角色与用户角色关联 |
| `balance_accounts`           | 用户余额账户       |
| `order_batches`              | 批量订单批次       |
| `orders`                     | 单条订单           |
| `account_records`            | 账户资金流水       |
| `note_basic_cache`           | 笔记信息缓存       |
| `open_api_keys`              | Open API 密钥      |
| `system_configs`             | 系统配置           |

## 8. 运维管理

### 8.1 PM2 常用命令

```bash
cd el_node

# 启动
npx pm2 start ecosystem.config.js

# 查看状态
npx pm2 status

# 查看日志
npx pm2 logs goosd-admin-api

# 重启
npx pm2 restart goosd-admin-api

# 停止
npx pm2 stop goosd-admin-api
```

### 8.2 日志文件

- PM2 日志：`el_node/logs/pm2-out.log`、`el_node/logs/pm2-error.log`
- 请求日志：`el_node/logs/requests-YYYY-MM-DD.log`（保留 2 天）

### 8.3 数据库备份

```bash
mysqldump -u goosd_admin -p goosd_admin > backup_$(date +%Y%m%d).sql
```

### 8.4 前端更新部署

```bash
cd webEL
pnpm build
# 将生成的 dist.zip 上传到服务器并解压到 Web 目录
```
